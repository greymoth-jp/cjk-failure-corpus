GROWI shows relative dates in English ("3 days" not "3日"): no locale passed to date-fns

Relative dates render in English for everyone (no locale passed to date-fns)

GROWI · growilabs/growi

Symptom

FormattedDistanceDate calls formatDistanceStrict without a locale, so relative times render in English regardless of the user's language. A Japanese user sees "3 days" instead of "3日" in comments, in-app notifications, and Recent Changes.

Minimal repro 
1. Set the UI language to Japanese.
2. Open page comments, in-app notifications, or the Recent Changes sidebar.
3. The relative timestamps read "3 days" instead of "3日".
Fix

Pass the resolved date-fns locale to formatDistanceStrict ({ locale: getLocale(i18n.language) }), the same approach RecentActivity/ActivityListItem already uses. English output is unchanged.
