BlockNote: input rules fire on the Enter that confirms an IME conversion (React)

Input rules fire on the Enter that confirms an IME composition

BlockNote · TypeCellOS/BlockNote

Symptom

The input-rules-on-Enter handler does not check event.isComposing, so it also runs on the Enter that commits an IME composition. A matching rule (for example the # heading rule) fires mid-conversion and the preventDefault disrupts the commit.

Minimal repro 
1. In a BlockNote editor, type # with no trailing space.
2. Switch to a Japanese IME and type some hiragana.
3. Press Enter to confirm a kanji candidate.
4. The heading input rule fires on the IME-commit Enter, so the candidate does not commit normally.
Fix

Return early from the inputRulesEnter handler while event.isComposing is true (keyCode 229), the same guard the editor's other Enter handlers already use.
