micro editor: command-bar autocomplete clips wide CJK suggestions (display width vs character count)

Command-bar autocomplete scrolls by character count, so wide CJK suggestions get clipped

micro · micro-editor/micro

Symptom

The info-bar autocomplete strip scrolls by character count (CharacterCountInString) while the renderer draws by display width (runewidth), so a suggestion containing full-width CJK characters can be left partly or fully off screen.

Minimal repro 
1. Trigger command-bar autocomplete with suggestions like ["日本語テスト", "ok", "end"] in a 14-column strip.
2. Select the second suggestion.
3. 日本語テスト is 6 characters but 12 display columns, so hscroll stays at 0 and ok is clipped at the right edge.
Fix

Measure the scroll offset with runewidth.StringWidth so it matches totalSize and the renderer. ASCII-only suggestions are unchanged.
