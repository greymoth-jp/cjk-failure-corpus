# CJK / Unicode Failure Corpus

A searchable reference of real CJK, IME, and Unicode/text-handling bugs in open-source libraries. For each one: a one-line symptom, a minimal repro, which library it hits, and the fix.

Live: https://greymoth-jp.github.io/cjk-failure-corpus

Most of these are the same handful of mistakes, made over and over, in libraries that work fine in English. A Japanese user presses Enter to confirm an IME conversion and the form submits early. A katakana ン loses its apostrophe and collides with a different word. A daiji numeral in a contract does not parse. None of it shows up unless you read the code with a Japanese keyboard in mind, which is the whole point of keeping a list.

## What's in it

Every entry maps to a real pull request. Titles, repos, URLs, and merge status are taken from the GitHub API, not written from memory. The categories so far:

- **IME composition** — the Enter that confirms a kana-to-kanji conversion also fires a submit/select handler. The fix is one guard (`event.isComposing`, or `keyCode 229`; in React, `event.nativeEvent.isComposing`). It is endemic: the corpus tracks it across React, Vue, Svelte, Angular, and headless component libraries.
- **Kana / romaji** — transliteration tables that drop or reverse kana. Round-trip is the oracle.
- **Width / normalization** — full-width to half-width, long-vowel marks, kana range boundaries.
- **Numerals** — kanji numbers, including the daiji forms used in legal and financial text.
- **Locale data** — missing or wrong `ja-JP` strings, including mistranslations that flip meaning.
- **Regex roundtrip / codegen escape** — characters that are special in one position leaking into generated output and breaking it.

The full dataset is in [`data/corpus.json`](data/corpus.json).

## CI fixtures

Finding a bug once is not the same as keeping it fixed. The repros here are turned into CI fixtures in a companion repo so the regressions can be caught automatically: [greymoth-jp/cjk-agent-fixtures](https://github.com/greymoth-jp/cjk-agent-fixtures).

## Build

Plain HTML, CSS, and a little JavaScript. No framework, no dependencies. The site is generated by one Node script from `data/corpus.json`.

```sh
node build.mjs
```

That writes `index.html`, one page per entry under `e/`, plus `sitemap.xml` and `robots.txt`. To preview locally, serve the folder with any static server, for example:

```sh
python -m http.server 8000
```

To add an entry, append an object to `entries` in `data/corpus.json` and rebuild. The build fails loudly if an entry is missing a field or points at something that is not a GitHub PR or issue, so the page can never claim a fix it cannot link to.

Search and filtering run client-side over the server-rendered list, so the page works with JavaScript disabled and every entry has a stable URL for linking and search engines.
